## 问题背景

在 **Flutter iOS** 开发中，我们经常会遇到以下场景：

- 只有一个 **iPhone** 设备用于真机调试
- 数据库使用本地 **SQLite** 文件存储
- 不同环境（开发/生产）的 **Bundle ID** 是相同的

在这种情况下，从 **App Store** 下载的正式版 App 和**开发环境**安装的 App 会相互覆盖，并且共用同一个 **SQLite** 文件，导致**数据混合**。

大部分时间我们使用**模拟器**调试，不会遇到这个问题。但当需要**真机调试**时，我们必须将两个环境**隔离**开来，避免数据混乱。

更重要的是，我们有时需要删除应用来测试初始化功能，如果数据混合就无法准确测试。

为了解决这个问题，我们需要为**开发环境**和**生产环境**设置不同的 **Bundle ID**，这样就可以在同一台手机上看到多个**不同环境**的相同 App。

这里我只展示两个，一个是**真实发布**的 **Bundle ID**，一个对应的是**开发环境**的 **Bundle ID**。

## 准备环境

在开始配置之前，请确保你已经准备好以下环境：

- • **MacBook**
- • **VSCode** 用来开发 Flutter
- • **Xcode** 用来打包
- • **iOS Simulator** 用来调试

## 配置步骤

下面我们开始设置 **Xcode**，配置不同环境对应不同的 **Bundle ID**。

### 1. 打开 Xcode 项目

首先打开我们的 **Flutter** 工程的 **iOS** 文件，使用 **Xcode** 打开。

我们可以看到最上面的正中间有一个 **Runner**。

<BlogImage client:visible src="https://blogassets.4real.ink/blog/999c520d26d8c01e9042721559f66ce7.webp" alt="" width={1600} height={1023} thumbhash="+vcFBIABVah2h4h3iHdCIBECEg==" />

这个在 **Xcode** 里面称为 **Scheme**。不同的 **Scheme** 会对应一个不同的打包逻辑，我们在 **Flutter build iOS** 的时候就可以通过指定 **Scheme** 的名字来实现不同的打包流程。

那我们如果不指定 **Scheme** 的话，它默认的就是第一个 **Runner**。

### 2. 配置 Build Configuration

接着左边点一下 **Runner** 这个 Menu，然后点击里面 **Project** 下面的 **Runner**，继续点击 **Info**。

<BlogImage client:visible src="https://blogassets.4real.ink/blog/c7d863fdc41dd91bae80ee6ee5a778a0.webp" alt="" width={1600} height={1023} thumbhash="+vcFBIABVZh2iIiHiHijQAMXNQ==" />

在 **Configuration** 模块我们可以看到 **Debug/Release/Profile** 分别对应不同环境下打包的一些参数。

<BlogImage client:visible src="https://blogassets.4real.ink/blog/5439012e618e57dcca63c61a592c84e7.webp" alt="" width={1324} height={458} thumbhash="/gcCAoCDUGqJZqh2ob1AyAk=" />

我们在开发的时候就是默认 **Debug**，我们在发布的话默认就是 **Release**。

目前这里的 **Debug**、**Release** 和 **Profile** 它们配置的 **Bundle ID** 都是一个，所以你在开发环境使用调试，如果你发布到 **App Store** 里面再下载下来，就会**覆盖**你开发环境安装的应用，反之同理。

接着我们在 **Configuration** 模块，点击**加号**，选择 **Duplicate Debug**、**Release**。

<BlogImage client:visible src="https://blogassets.4real.ink/blog/55d07d50c5ada9330ef514a5d0daa137.webp" alt="" width={1600} height={1023} thumbhash="+vcFBIABVZh2iHiHd3iDQBIEFA==" />

然后我们再给它们分别改名，改叫 **Debug_dev**、**Release_dev**。

### 3. 设置 APP DISPLAY NAME

接着我们往左边看，看到 **Project** 下面有一个 **Targets**，Targets 我们依旧点击 **Runner**。

点击 **Runner** 之后我们可以看到上面的 **Tab** 有很多东西，接着我们点击 **Build Settings**。

<BlogImage client:visible src="https://blogassets.4real.ink/blog/cc3c036241eea41f7374396a50bf9039.webp" alt="" width={1600} height={1023} thumbhash="+vcFBIABVah2iIiHh3jDQDQJBA==" />

接着点击**加号**，然后添加一个环境变量 **APP_DISPLAY_NAME**，根据不同 **Build Configuration**，设置不一样的名字。

<BlogImage client:visible src="https://blogassets.4real.ink/blog/e88a66b01efc4f8c5db85cd214d8ae16.webp" alt="" width={1600} height={1023} thumbhash="+fcFBIAAVZd2iHiHh4eSUAUTVg==" />

<BlogImage client:visible src="https://blogassets.4real.ink/blog/4baec8e7021812d85dfb2c41f15d5516.webp" alt="" width={1600} height={1023} thumbhash="+fcFBIAARJd2iHh4h3fZhW8b+A==" />

**注意**这里只是注册了一个**变量**，并不会真正改变打包后的应用名字，我们还需要在 **info.plist** 里面进行修改。

现在我们在 **Xcode** 里面打开 **Info.plist**，然后找到 **Bundle display name**，修改其 **value** 如图所示，改成刚刚添加的变量名 **$(APP_DISPLAY_NAME)**

<BlogImage client:visible src="https://blogassets.4real.ink/blog/ae02c9b5598cdcdba94ba848bff90c9f.webp" alt="" width={1600} height={1023} thumbhash="+vcFBIAAZZh2iIiYiIgQZx6Axg==" />

### 4. 配置 Bundle Identifier

我们现在开始为不同的配置设置不同的 **Bundle ID**。点击旁边的 **Targets** → **Runner**，然后点击 **General**。

<BlogImage client:visible src="https://blogassets.4real.ink/blog/7c1595025c08f5b757aa6889583ee903.webp" alt="" width={1600} height={1023} thumbhash="+vcFBIABVZh3h3h3iHcyIDAhAw==" />

之后我们再去看到下面有个 **Identity**，我们看到有个 **Bundle Identifier**，然后旁边有一个**箭头**，我们点击一下。

<BlogImage client:visible src="https://blogassets.4real.ink/blog/1f5c38990a03e48cef6ebde9994d45b9.webp" alt="" width={1600} height={1023} thumbhash="+vcFBIABVZh3h3h3iHchIAAQBA==" />

点进去后，会看到有不同的 **Bundle Identifier** 配置项，我们为每个环境填入**不同的** **Bundle ID** 即可。

<BlogImage client:visible src="https://blogassets.4real.ink/blog/144c45dd55e7e2f8a27a3e7bafac3898.webp" alt="" width={1600} height={1023} thumbhash="+vcFBIABRah2h4h3h3cxEAEBIg==" />

### 5. 创建新的 Scheme

**Flutter** 可以通过 **--flavor** 命令来调用不同的 **Scheme**，也就是我们最开始说过的。

我们只需要创建一个新的 **Scheme** 来对应新的 **Configuration** 就好了。

我们点击 **Manage Scheme**。

<BlogImage client:visible src="https://blogassets.4real.ink/blog/87fa6aca0a3c83a8da679cbadebf21bf.webp" alt="" width={1600} height={1023} thumbhash="+fcFBIAAVZh2iHh3iHfVUAAmGA==" />

点击右下角的**加号**。

<BlogImage client:visible src="https://blogassets.4real.ink/blog/1895cd79ce29c8916e1c887ff1a1a9e0.webp" alt="" width={1600} height={1023} thumbhash="8/cFBIAFZqh3iIeCenjodY9O9w==" />

选择默认的 **Runner**，然后我们输入新的名字，比如 **dev**，然后点击 **OK** 创建。

<BlogImage client:visible src="https://blogassets.4real.ink/blog/77107cf3a51be4e0a36a5d48fbb4caf0.webp" alt="" width={1536} height={858} thumbhash="8/cFBIBHd5iIh3dwi4Wbee+n9w==" />

然后我们退出来之后，我们再点击 **Runner**，点击 **Edit Scheme**。

<BlogImage client:visible src="https://blogassets.4real.ink/blog/06b5753f5c4476d86e93379ffcc856ad.webp" alt="" width={1600} height={1023} thumbhash="+fcFBIAARZh2iHh3iHfEUAEVFQ==" />

我们先选择我们刚刚创建的 **Scheme**。

<BlogImage client:visible src="https://blogassets.4real.ink/blog/7b25e4255ae8b76e422cec8a82a4fb05.webp" alt="" width={1600} height={864} thumbhash="/PcFBIAwdYhleIl2iHpKBNRhQA==" />

接着我们点击左边的 **Run**，**Build Configuration** 改成 **Debug_dev**。

<BlogImage client:visible src="https://blogassets.4real.ink/blog/7c4ed99e99ec9143833c03b12b727c2c.webp" alt="" width={1600} height={864} thumbhash="/PcFBIAwU5h2aIeIh3kXBrWSUA==" />

下面的 **Archive** 也是一样的，切换成 **Release_dev**。

<BlogImage client:visible src="https://blogassets.4real.ink/blog/567900d459d61c61aa40c91a089cb084.webp" alt="" width={1600} height={864} thumbhash="/PcFBIAgVJh3eHeXiHgDXE3Q8g==" />

到现在为止，**Xcode** 的配置全部完毕。

## VSCode 配置不同的启动命令

为了方便在开发时切换不同的环境，我们需要在 **VSCode** 中配置不同的启动命令。

找到 **VSCode** 的 **Debug** 页面，点击这个**设置按钮**开始配置运行命令。

<BlogImage client:visible src="https://blogassets.4real.ink/blog/9e5cd6bfa914b05c091bb932e82fadc3.webp" alt="" width={1600} height={949} thumbhash="wvcBBIAqkMtEmWavlmi3r5b6ag==" />

可以直接复制以下**配置**：

```json
{
  // Use IntelliSense to learn about possible attributes.
  // Hover to view descriptions of existing attributes.
  // For more information, visit: https://go.microsoft.com/fwlink/?linkid=830387
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Flutter Dev",
      "request": "launch",
      "type": "dart",
      "args": [
        "--flavor",
        "dev",
        "-t", // if you have different main file
        "lib/main.dart"
      ]
    },
    {
      "name": "Flutter Normal",
      "request": "launch",
      "type": "dart",
      "args": [
        "-t", // if you have different main file
        "lib/main.dart"
      ]
    }
  ]
}
```

解释一下命令 `--flavor dev`，这个是用来**区分不同环境**的，**dev** 对应的是 **Scheme** 的名字，如果没有指定就是默认的 **Runner**。

## 打包命令

在**打包**的时候记住如果要区分也要指定 `--flavor dev`：

```typescript
flutter build ios --release --flavor dev
```

## 总结

通过以上配置，我们成功实现了在同一台 **iOS** 设备上安装**不同环境**的同一应用。**主要步骤**包括：

1. 1. 在 **Xcode** 中创建不同的 **Build Configuration**
2. 2. 设置不同的 **Bundle ID** 和**应用显示名称**
3. 3. 创建对应的 **Scheme**
4. 4. 在 **VSCode** 中配置**启动命令**

通过这种方式，我们就能有效避免**开发环境**和**生产环境**的**数据混合问题**，在同一台设备上安全地进行**多环境测试**。
